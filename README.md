# Favorita Grocery Sales Forecasting

## Описание

Решение задачи прогнозирования продаж для соревнования [Corporación Favorita Grocery Sales Forecasting](https://www.kaggle.com/competitions/favorita-grocery-sales-forecasting) на Kaggle.

**Метрика**: NWRMSLE (Normalized Weighted Root Mean Squared Logarithmic Error)

## Данные

Датасет расположен по пути `/kaggle/input/datasets/nikolasking/favorita/`.

Файлы:
| Файл | Описание |
|------|----------|
| `train.csv` | Продажи по дате, магазину, товару (2013–2017) |
| `test.csv` | 16-дневный тестовый горизонт (авг 2017) |
| `stores.csv` | Метаданные магазинов (город, штат, тип, кластер) |
| `items.csv` | Метаданные товаров (семейство, класс, скоропортящийся) |
| `transactions.csv` | Количество транзакций по дате и магазину |
| `oil.csv` | Цены на нефть (экзогенный признак) |
| `holidays_events.csv` | Праздники и события в Эквадоре |

---

## Описание подхода

### Постановка задачи
Предсказать ежедневные продажи (`unit_sales`) для каждой пары (магазин, товар) на 16 дней вперёд.

### Прогнозный период
Time-based split: последние 16 дней обучающей выборки (01–15 авг 2017) используются как локальная валидация, имитируя тестовый горизонт.

### Генерация признаков (Feature Engineering)
Для улучшения предсказательной способности моделей были созданы следующие группы признаков:
* **Временные признаки**: День недели, месяц, флаги выходных и праздничных дней (is_holiday).
* **Авторегрессионные лаги**: Использование продаж за 16 и 21 день до текущей даты (`lag_16`, `lag_21`).
* **Окна агрегации**: Скользящее среднее за последние 28 дней (`rolling_mean_28`) для улавливания среднесрочных трендов.
* **Внешние данные**: Динамика цен на нефть (`dcoilwtico`) с лагом в 1 день, интегрированная из файла `oil.csv`.
* **Промо-акции**: Лагированные признаки активности промо-кампаний (`promo_lag_16`).

### Бейзлайны
1. **Mean Baseline**  — прогноз = среднее значение продаж за последние 14 дней
2. **Seasonal Naive*** — прогноз = значение той же недели (сезонность 7 дней)

### ML-модель: CatBoost
    * Использование категориальных признаков без предварительного One-Hot Encoding.
    * Обучение на GPU для ускорения обработки 20 млн строк.
    * Анализ Feature Importance для интерпретации весов признаков.

### DL-модель: LSTM
    * Архитектура: Stacked LSTM с полносвязным декодером.
    * Embeddings: Обучаемые векторные представления для магазинов, семейств товаров и кластеров.
    * Sequence length: 30 дней; Forecast horizon: 15 дней.
    * Оптимизатор: Adam с градиентным клиппингом (1.0).

## Результаты и метрики


| Модель | NWRMSLE (Val) | Комментарий |
|--------|---------|-------------|
| **Mean Baseline** | 0.5689 | Базовое среднее по серии |
| **Naive** | 0.8120 | Прогноз по последнему дню |
| **CatBoost** | 0.5232 | Лучшая интерпретируемость |
| **LSTM** | **0.5212** | **Best Single Model** |


## Анализ ошибок
* **Средняя ошибка (Mean Residual)**: 0.03. Модель демонстрирует отсутствие системного смещения, сохраняя легкий консервативный недопрогноз (~3%) в моменты экстремальных пиков спроса.
* **Взвешенная метрика**: LSTM показала результат NWRMSLE (0.5212) ниже, чем RMSLE (0.5225), что подтверждает высокую точность прогнозирования скоропортящихся товаров (perishable), имеющих больший вес в итоговой оценке.
* **Слабые места**: Основные ошибки сконцентрированы в товарах с коротким жизненным циклом и аномальных всплесках продаж, не связанных с календарем праздников или промо-акций.
